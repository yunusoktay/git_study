# Day 001

## Focus

- Set up the Dart environment
- Write a simple `Hello, World!` program

## Run


## Outcome

Day 1 gives a simple start for Dart. It shows how to use variables and basic data types.

`final` and `const` are both for values that do not change, but they are not the same.

`final` means a variable gets its value one time. This value can be known when the program is running.

`const` means the value must be known before the program runs.

The main difference is when the value is known.

- `final`: runtime constant
- `const`: compile-time constant

`compile time` is the time before the program runs. In this step, Dart checks and prepares the code.

`runtime` is the time when the program is running.

`const` is connected to compile time because its value must be ready early. `final` can be connected to runtime because its value can be set while the program runs, but only one time.

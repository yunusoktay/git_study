# Day 003

## Focus

- Understand sound null safety
- Work with nullable types
- Use null-aware operators
- Learn `!` and `late`

## Run

From the repository root:

```bash
dart run dart/day_003.dart
```

## Outcome

Day 3 is about null safety. Null safety helps you avoid errors when a value is `null`.

In Dart, types are non-nullable by default. This means a variable should have a real value, not `null`.

A nullable type uses `?`. For example, `String?` means the variable can hold a string or `null`.

The `?.` operator lets you use a property or method only when the value is not `null`.

The `??` operator gives a default value when the first value is `null`.

The `!` operator means you tell Dart that a nullable value is not `null` at that moment. It should be used carefully.

The `late` keyword means a non-nullable variable will get its value later, before it is used.

# Day 008

## Focus

- Understand abstract classes
- Create abstract methods
- Understand interfaces in Dart
- Use the `implements` keyword

## Run

From the repository root:

```bash
dart run dart/day_008.dart
```

## Outcome

Day 8 introduces abstract classes and interfaces.

The example uses:

- `Shape` as an abstract class with an abstract `area()` method and a concrete `describe()` method
- `Circle` and `Rectangle` extend `Shape` and provide their own `area()` implementation
- `Logger` as an interface-style abstract class
- `ConsoleLogger` implements `Logger`

## Key Concepts

| Concept | Where in the Code | What It Means |
|---|---|---|
| **Abstract class** | `abstract class Shape` | A class that cannot be instantiated, meant to be a base |
| **Abstract method** | `double area();` | Method without a body, subclass must implement it |
| **Concrete method** | `void describe() { ... }` | Regular method inside an abstract class, inherited as-is |
| **`extends`** | `class Circle extends Shape` | Inherits from abstract class and fills in abstract methods |
| **`implements`** | `class ConsoleLogger implements Logger` | Fulfills the contract of an interface |
| **Polymorphism** | `List<Shape> shapes = [circle, rectangle]` | Different objects handled through the same abstract type |

## extends vs implements

- `extends` → inherits code (properties + method bodies) from the parent
- `implements` → only takes the shape (signatures), you must rewrite everything yourself

## Why Use Them

- **Abstract class** is useful when you want a shared base with some default behavior but force subclasses to fill in specific parts.
- **Interface** is useful when you want to define a contract that many unrelated classes can follow without sharing code.

In Dart, every class implicitly defines an interface, so any class can be used after `implements`. A class can only `extends` one class, but can `implements` multiple classes.

# Day 012

## Focus

- Create a `Future` using `Future.delayed`
- Handle success with `.then(...)`
- Handle errors with `.catchError(...)`
- Always-run callback with `.whenComplete(...)`
- Chain multiple `.then(...)` calls

## Run

From the repository root:

```bash
dart run dart/day_012.dart
```

## Outcome

Day 12 shows the three core Future methods together:

- `.then()` fires when the Future **succeeds**
- `.catchError()` fires when the Future **fails**
- `.whenComplete()` fires **either way** (like `finally`)

### Example structure

```dart
fetchUser(true)
    .then((user) => print(user))
    .catchError((error) => print(error))
    .whenComplete(() => print('done'));
```

### Chaining then()

A `.then(...)` can return a new value, and the next `.then(...)` receives that value:

```dart
fetchUser(true)
    .then((user) => 'Welcome, $user')
    .then((greeting) => print(greeting));
```

## Key Concepts

| Method | When it runs |
|---|---|
| **`.then(callback)`** | Future completes with a value |
| **`.catchError(callback)`** | Future completes with an error |
| **`.whenComplete(callback)`** | Always runs at the end (success or error) |

## try-catch-finally Karsiligi

Sync kod icin:

```dart
try {
  // work
} catch (e) {
  // error
} finally {
  // always
}
```

Future icin:

```dart
work()
    .then((v) => ...)
    .catchError((e) => ...)
    .whenComplete(() => ...);
```
